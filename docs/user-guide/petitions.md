# Petitions

Petitions are formal expressions of community will on CivicNotes. They transform individual concerns into collective demands, gathering verified signatures to demonstrate widespread support for a specific action. When a petition reaches its goal, it automatically creates a Promise Tracker record, putting direct pressure on the target body to respond.

This guide covers everything you need to know about petitions—from creation to celebration.

---

## What Is a Petition?

A **petition** is a formal request, signed by verified community members, asking a government body (e.g., City Council, School Board) to take a specific action. Petitions on CivicNotes are deeply integrated with the platform’s accountability tools.

Each petition includes:
- **Title and description** – What you're asking for
- **Target body** – Who has the power to act
- **Signature goal** – How many verified signatures you need
- **Deadline** – When the petition closes
- **Current signature count** – Real‑time progress
- **Status** – Active, Threshold Met, Submitted, Acknowledged, Addressed, Expired

---

## Why Petitions Matter

Petitions on CivicNotes are different from traditional paper or generic online petitions:

| Traditional Petitions | CivicNotes Petitions |
|----------------------|----------------------|
| Signatures often unverified | Signatures verified via SMS, email, or in‑person |
| No direct link to government process | Auto‑creates Promise Tracker record when goal met |
| Easy to ignore | Visible to officials in their dashboard |
| Disappear after submission | Lifecycle tracked publicly |
| No connection to other civic engagement | Built from CivicNotes, proposals, or regenerative actions |

Petitions are a key **dual‑power** tool—they demonstrate organized community consent and create legitimate pressure for change.

---

## Petition Lifecycle

A petition moves through these stages:
```markdown
Draft → Active → Threshold Met → Submitted → Acknowledged → Addressed
↘ Expired
```

| Status | Description |
|--------|-------------|
| **Draft** | Not yet public; creator is still editing. |
| **Active** | Open for signatures; visible in petitions feed. |
| **Threshold Met** | Reached signature goal; will be submitted automatically. |
| **Submitted** | Formally delivered to target body; Promise Tracker record created. |
| **Acknowledged** | Target body has publicly responded. |
| **Addressed** | Action has been taken (verified). |
| **Expired** | Deadline passed without meeting goal. |

## Creating a Petition

### When to Create a Petition

Good times to start a petition:
- A CivicNote on an agenda item gains significant support (many likes, replies)
- Your guild wants to push a specific proposal
- You've identified a regenerative action that needs community backing
- An issue is urgent and needs broad visibility before an upcoming vote

### Step 1: Find Your Starting Point

You can create a petition from:

| Starting Point | How |
|----------------|-----|
| **CivicNote** | On any note, click the "..." menu and select "Start Petition." The note's text will pre‑fill as the petition description. |
| **Proposal** | From a proposal page, click "Launch Petition." The proposal text will be used. |
| **Regenerative Action** | After adapting an action, choose "Launch Petition" as the output. |
| **From scratch** | Go to the Petitions page and click "Start New Petition." |

### Step 2: Fill Petition Details

You'll be guided through a multi‑step form.

**Basic Information**
```markdown
+------------------------------------------------------------------+
| CREATE PETITION · STEP 1 OF 4 |
| |
| Title: [_______________________________________________] |
| |
| Description: [________________________________________] |
| (Be clear and specific about what you're asking for.) |
| |
| [Continue] |
+------------------------------------------------------------------+
```

**Target & Goal**
```markdown
+------------------------------------------------------------------+
| CREATE PETITION · STEP 2 OF 4 |
| |
| Target Body: [City Council ▼] |
| |
| Specific Committee (optional): [Transportation Committee] |
| |
| Signature Goal: [______] signatures |
| |
| Deadline: [📅 Date Picker] |
| |
| [Continue] |
+------------------------------------------------------------------+
```

**Signature Verification** – Choose how signers will verify their identity:
- **SMS** (fastest, requires phone number)
- **Email** (good for online‑only campaigns)
- **In‑person** (for events; generates QR codes or printable sheets)
- **Any combination** – You can allow multiple methods.

**Accountability Settings**
```markdown
+------------------------------------------------------------------+
| CREATE PETITION · STEP 3 OF 4 |
| |
| When goal is met: |
| ☑ Automatically create Promise Tracker record for target body |
| ☑ Notify all signers |
| ☑ Notify relevant guilds |
| |
| [Continue] |
+------------------------------------------------------------------+
```

**Review & Launch**
```markdown
+------------------------------------------------------------------+
| CREATE PETITION · STEP 4 OF 4 |
| |
| REVIEW YOUR PETITION |
| |
| Title: Safe Streets at 5th & Main |
| Target: City Council · Transportation Committee |
| Goal: 1,000 signatures by June 30, 2025 |
| Verification: SMS and email |
| |
| Description: |
| We demand the City Council fund a traffic calming study and |
| community design process for the intersection at 5th and Main... |
| |
| [Edit] [Launch Petition] |
+------------------------------------------------------------------+
```

Once launched, your petition appears in the **Active Petitions** feed and on the relevant agenda item page (if linked).

## Signing a Petition

### Finding Petitions to Sign

- **Your dashboard** – Shows petitions trending in your neighborhood.
- **Petitions tab** – Browse all active petitions, filter by target, district, or topic.
- **Agenda items** – Related petitions appear on the item page.
- **Guild feeds** – Guilds you follow may endorse petitions.

### The Signing Process

1. Click **"Sign"** on any active petition.
2. Choose your verification method:
   - **SMS** – Enter your phone number; receive a code; enter it to confirm.
   - **Email** – Receive a confirmation link; click it.
   - **In‑person** – Show a QR code at an event, or staff will scan your sheet.
3. (Optional) Choose whether your name appears publicly.
4. Click **"Confirm Signature"**.
```markdown
+------------------------------------------------------------------+
| SIGN THIS PETITION |
| |
| Safe Streets at 5th & Main |
| Target: City Council · 1,000 signatures needed |
| |
| Verify your signature: |
| ○ SMS (fastest) |
| ○ Email |
| ○ I'll sign in person (show QR code at event) |
| |
| ☑ Add my name to public supporter list |
| ☐ Keep my signature private (counts but name hidden) |
| |
| [Confirm Signature] |
+------------------------------------------------------------------+
```

After signing, you'll receive a confirmation and can share the petition with your network.

## When a Petition Reaches Its Goal

Once the signature threshold is met, the following happens automatically:

1. **Status changes** to **Threshold Met**.
2. A **Promise Tracker record** is created for the target body, with the petition as the source.
3. **Notifications** are sent to:
   - All signers
   - The petition creator
   - Endorsing guilds
   - The target body's official dashboard
4. The petition page now shows **"Submitted"** status and a link to the Promise Tracker record.

The target body now has a verified, public demand waiting for response.

---

## After Submission

### Official Response

Officials can respond directly from their dashboard:
- **Acknowledge** – Publicly recognize the petition.
- **Add comment** – Provide update (e.g., "This will be on next month's agenda").
- **Mark as Addressed** – If action taken, with evidence.

All responses appear on the petition page.

### If No Response

If the target body doesn't respond within a reasonable time (e.g., 60 days), the petition remains as a public record. Community members can:
- **Escalate** – Share the unfulfilled petition widely.
- **Use in advocacy** – Reference it in CivicNotes, at meetings, or in media.
- **Re‑launch** – Start a new petition with higher goal or broader coalition.

### When Addressed

When the requested action is taken, anyone can propose marking the petition **Addressed** (similar to promises). Provide evidence (resolution link, photo, etc.) for community verification. Once verified:
- Status becomes **Addressed** (green).
- Φ‑Score increases for the relevant district and agenda item.
- All signers receive a celebration notification.
- Civic Seeds awarded to key contributors.

## Petitions and the Φ‑Score

Petitions contribute to multiple Φ‑Score sub‑metrics:

| Action | Φ‑Score Impact |
|--------|----------------|
| New petition launched | +1 to Participation (per 100 goal) |
| Signature added | +0.1 to Participation |
| Petition reaches goal | +3 to Participation, +2 to Responsiveness |
| Official acknowledges | +5 to Responsiveness |
| Petition addressed | +10 to Responsiveness, +3 to Connections |

Petitions also appear in the **Connections** metric by showing cross‑neighborhood and cross‑guild support.

## Tips for Effective Petitions

### Writing a Strong Petition

- **Clear ask** – State exactly what you want the body to do (e.g., "Adopt a resolution to..." rather than "Do something about...").
- **Local specifics** – Include addresses, project names, and deadlines.
- **Connect to existing process** – Reference an upcoming agenda item if possible.
- **Use regenerative language** – Frame your ask as a solution, not just a complaint.

### Building Momentum

- **Start with a core group** – Get your guild or close network to sign first.
- **Set a realistic goal** – 500–1,000 is often enough to get attention; adjust for your community size.
- **Promote everywhere** – Share on social media, at meetings, through email lists.
- **Update signers regularly** – Use the message tool to keep people engaged.
- **Coordinate with other efforts** – Align with related petitions or proposals.

### Verification Choices

- **SMS** is fastest and most secure – use as primary method.
- **Email** is good for broad reach but slightly lower verification rate.
- **In‑person** is excellent for events and building real‑world connections.

## Petitions and Guilds

Guilds play a special role in petitions:

- **Endorsement** – A guild endorsement adds a badge and increases visibility. Guild‑endorsed petitions appear in guild feeds and may get promoted.
- **Co‑sponsorship** – Multiple guilds can co‑sponsor a petition, showing broad coalition support.
- **Verification weight** – Signatures from guild members may be weighted slightly higher in analytics (not in count, but in influence metrics).
- **Guild‑led petitions** – Guilds can create petitions directly from their dashboard and track them collectively.

## Frequently Asked Questions

### Can anyone start a petition?
Yes, any verified CivicNotes user can start a petition.

### How are signatures verified?
By SMS code, email confirmation link, or in‑person QR code/paper sheet. This prevents fraud and ensures each signature represents a real person.

### What if the target body ignores the petition?
The petition remains as a public record. You can use the Promise Tracker to track non‑response, and the lack of acknowledgment will affect the body's responsiveness Φ‑Score.

### Can I change the goal after launch?
No, goals are fixed to maintain integrity. You can, however, create a new petition with a higher goal if needed.

### What happens to signatures if the petition expires?
Signatures are still visible, and you can download the list. The petition status becomes Expired, but you can relaunch it.

### Can officials sign petitions?
Officials can sign as private citizens, but their signature is clearly marked with an "Official" badge. They cannot sign petitions targeting their own body.

### How do I download signatures?
From your petition dashboard, click "Export Signers" to get a CSV file (includes name, neighborhood, verification method, date).

### Can I edit the description after launch?
Minor edits (typos, clarifications) are allowed. Major changes that alter the ask require a new petition.
