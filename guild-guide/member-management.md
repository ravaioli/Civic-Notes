# Member Management

Your guild is only as strong as its members—and how you welcome, support, and organize them makes all the difference. This guide covers everything you need to know about managing your guild's membership: inviting new people, approving requests, assigning roles, handling departures, and keeping everyone engaged.

---

## Member Lifecycle Overview
```markdown
Discovery → Application → Approval → Onboarding → Active Participation → (Optional) Role Advancement → (Optional) Departure
```

| Stage | Description |
|-------|-------------|
| **Discovery** | Potential members find your guild through search, recommendations, or invitations. |
| **Application** | They request to join (if your guild requires approval) or join immediately (if open). |
| **Approval** | Admins review and approve/decline requests. |
| **Onboarding** | New members are welcomed and introduced to guild tools and culture. |
| **Active Participation** | Members engage in discussions, votes, endorsements, and proposals. |
| **Role Advancement** | Active members may be invited to take on leadership roles. |
| **Departure** | Members leave voluntarily or are removed for inactivity or Code of Conduct violations. |

---

## Member Roles and Permissions

Your guild can have members with different levels of responsibility. Roles are assigned from the **Members** tab in your guild dashboard.

| Role | Privileges | Typical Members |
|------|------------|-----------------|
| **Member** | • View guild content<br>• Participate in discussions<br>• Vote on internal decisions (if enabled)<br>• Suggest endorsements | Most guild members |
| **Endorsement Manager** | • All Member privileges<br>• Review endorsement suggestions<br>• Cast the guild's endorsement vote<br>• Approve/decline endorsements | Active members trusted with strategic decisions |
| **Admin** | • All Endorsement Manager privileges<br>• Manage members (approve/remove)<br>• Edit guild profile and settings<br>• Create announcements<br>• Represent guild in Guild Council (if designated) | Founding members, elected leads |
| **Founder** | • All Admin privileges<br>• Transfer guild ownership<br>• Delete guild (rare) | Original applicant(s) |

> **Note:** A member can hold multiple roles (e.g., Admin and Endorsement Manager). Roles are additive.

## Step 1: Setting Membership Rules

Before inviting members, decide how people can join your guild. Go to **Settings > Membership Rules**.
```markdown
+------------------------------------------------------------------+
| MEMBERSHIP RULES |
| |
| ● Open – Anyone can join immediately |
| (No approval needed; members added automatically) |
| |
| ○ Approval required – New members must be approved by an admin |
| (Recommended for most guilds) |
| |
| ○ Invite‑only – Only invited users can join |
| (Good for small, focused groups) |
| |
| ○ Closed – Not accepting new members at this time |
| (Existing members remain, but no new applications) |
| |
| [Save Changes] |
+------------------------------------------------------------------+
```

### Choosing the Right Model

| Model | Best For |
|-------|----------|
| **Open** | Broad‑based community guilds (e.g., "Environmentally Concerned Residents"). Low barrier to entry, but may require more moderation. |
| **Approval required** | Most guilds. Balances accessibility with quality control. |
| **Invite‑only** | Steering committees, working groups, or guilds in formation. |
| **Closed** | Guilds that have reached capacity or are winding down. |

## Step 2: Inviting New Members

### Method A: Shareable Invite Link

From your guild dashboard, click **"Invite"** in the Members tab. You'll get a unique URL:
```markdown
https://civicnotes.org/join/guild/safe-streets/abc123
```

- Share this link via email, social media, or at community events.
- Anyone who clicks it will be taken to your guild page with a pre‑filled join request (or automatic addition, depending on your rules).
- Links can be reset if compromised.

### Method B: Direct Invitations by Username

1. From the Members tab, click **"Invite by Username."**
2. Enter the CivicNotes username of the person you want to invite.
3. Add an optional personal message.
4. Send. The user receives a notification and can accept.

### Method C: Bulk Import (Future Feature)

For large existing organizations, a CSV upload feature may be available to invite multiple members at once.

## Step 3: Handling Join Requests

If your guild requires approval, join requests appear in your dashboard's **Pending Actions** and the Members tab.

```markdown
| |
| +----------------------------------------------------------------+
| | Taylor (@taylor) · District 3 · requested Apr 20 |
| | Reason for joining: "I walk my kids past 5th & Main every |
| | day and want to help make it safer." |
| | [Approve] [Decline] [Message] |
| +----------------------------------------------------------------+
| | Jordan (@jordan) · District 5 · requested Apr 19 |
| | Reason for joining: "Interested in traffic calming." |
| | [Approve] [Decline] [Message] |
| +----------------------------------------------------------------+
| | Casey (@casey) · District 2 · requested Apr 18 |
| | (No reason provided) |
| | [Approve] [Decline] [Message] |
| +----------------------------------------------------------------+
+------------------------------------------------------------------+
```

### Reviewing Requests

Consider:

- **Is the member verified?** (Look for the "Verified Resident" badge on their profile.)
- **Does their stated interest align with your guild's mission?**
- **Have they been active on CivicNotes?** (Check their note history.)
- **Do you recognize them from other community spaces?** (Optional, but helpful.)

### Responding

- **Approve** – They become a member immediately and receive a welcome notification.
- **Decline** – Optional: send a brief, kind reason (e.g., "We're currently focused on District 3 issues, but thank you for your interest.").
- **Message** – Ask a clarifying question before deciding.

> **Tip:** Respond to requests within a few days. Long‑pending requests can discourage potential members.

## Step 4: Onboarding New Members

First impressions matter. A good onboarding experience sets the tone for engagement.

### Automated Welcome

When a member joins, they automatically receive:

- A welcome notification with a link to your guild's charter and key discussions.
- An invitation to introduce themselves in a designated **"Introductions"** thread (if your guild has one).
- A badge on their profile showing guild membership.

### Suggested Onboarding Practices

| Practice | Description |
|----------|-------------|
| **Personal welcome** | An admin or designated "welcome buddy" sends a direct message within a few days. |
| **Orientation document** | A pinned announcement or discussion with FAQs, key links, and upcoming events. |
| **New member thread** | A space where new folks can introduce themselves and be greeted by existing members. |
| **Mentor pairing** | For larger guilds, pair new members with experienced ones for their first month. |
| **First action suggestion** | "Now that you've joined, here are three notes that need endorsement votes!" |

### Sample Welcome Message

> Hi [Name], welcome to the Safe Streets Guild! We're glad you're here.
>
> To get started:
> - Read our [charter](link) to understand our mission.
> - Introduce yourself in the [Introductions thread](link).
> - Check out [this week's endorsement queue](link) – we'd love your input.
> - Our next guild meeting is [date/time] – [RSVP here](link).
>
> If you have questions, just reply to this message. Looking forward to working together!

- **Host events** – Regular meetings, social hours, or working sessions build connection.
- **Ask for input** – "What issues should our next proposal address?" surveys.

## Step 6: Role Advancement

As members become more active, consider inviting them to take on additional responsibility.

### Endorsement Manager

Look for members who:
- Regularly suggest high‑quality endorsements.
- Participate thoughtfully in endorsement discussions.
- Understand your guild's focus and values.

**Process:** An admin can change their role to "Endorsement Manager" from the Members tab.

### Admin

Promote to Admin when a member demonstrates:
- Consistent, responsible engagement over many months.
- Good judgment in handling sensitive situations.
- Availability and willingness to help with administrative tasks.
- Alignment with guild values and Code of Conduct.

> **Recommendation:** Have at least two admins to share the workload and ensure continuity.

### Leadership Rotation

Some guilds prefer rotating leadership to prevent burnout and build shared capacity. Consider:

- Term limits for admins (e.g., 1 year, renewable).
- Elections for certain roles.
- A "stewardship circle" model with rotating facilitators.

## Step 7: Handling Member Departures

### Voluntary Departure

Members can leave a guild at any time from their **Guilds** page. When they leave:

- They are removed from the member list.
- Their past contributions (comments, votes, endorsements) remain visible but are no longer associated with an active member (displayed as "[Former Member]").
- Admins are not notified automatically (unless the departing member chooses to send a message).

### Involuntary Removal

Sometimes removal is necessary:

| Reason | Process |
|--------|---------|
| **Inactivity** | After 6 months of no activity, an admin may remove the member. Consider sending a warning first. |
| **Code of Conduct violation** | Document the violation, discuss with other admins, and remove. The member can appeal to the Moderation Council. |
| **Disruptive behavior** | Similar to Code of Conduct violations – document, discuss, act. |

**How to remove a member:**
1. Go to the Members tab.
2. Find the member in the list.
3. Click the "···" menu and select "Remove from Guild."
4. (Optional) Send a brief explanation (kind but firm).

Removed members can reapply in the future, but past behavior will be considered.

## Step 8: Member Settings and Preferences

### Member‑Visible Settings

From their end, members can control:

- **Notification preferences** – Which guild activities trigger alerts.
- **Privacy** – Whether their membership appears on their public profile.
- **Display name** – How they appear in guild contexts (can differ from main profile name).

### Admin‑Controlled Settings

From **Settings**, admins can configure:

- Whether members can invite others (useful for growth).
- Minimum account age/days on platform to join.
- Whether members can see each other's contact info.
- Default notification settings for new members.

## Best Practices

### Do's

- **Welcome promptly** – A quick welcome sets a positive tone.
- **Communicate expectations** – Be clear about what membership means (activity levels, Code of Conduct).
- **Recognize contributions** – Public thanks go a long way.
- **Check in with inactive members** – A simple "We miss you, everything okay?" can re‑engage.
- **Document decisions** – Especially for removals or role changes, keep internal notes.

### Don'ts

- **Let requests pile up** – Check pending approvals regularly.
- **Ignore conflicts** – Small disagreements can become big problems if unaddressed.
- **Promote without discussion** – Role changes should be considered, not impulsive.
- **Over‑administer** – Trust your members; you don't need to monitor every interaction.

The way you manage members shapes your guild's culture.

| If You Want... | Try... |

| **A welcoming, inclusive culture** | Active onboarding, new member threads, buddy system. |

| **Efficient, focused action** | Clear roles, streamlined approval processes. |
| **Long‑term sustainability** | Leadership rotation, documentation of processes. |

## Frequently Asked Questions

### How many members should we have?
There's no ideal number. Some guilds thrive with 10 deeply engaged members; others with 100+ less active but broadly supportive members. Focus on alignment and engagement, not size.

### Can someone be in multiple guilds?
Absolutely. Many members belong to several guilds reflecting different interests (e.g., Safe Streets + Community Garden Guild). Their profiles show all memberships.

### What if a member is disruptive but not violating Code of Conduct?
Try addressing it directly and privately first. Often people don't realize their impact. If it continues, discuss with other admins and consider setting clear expectations or, as a last resort, removal.

### Can we have non‑resident members?
Guilds can choose. Some may want to include allies from neighboring districts. However, only verified residents can vote in certain platform‑wide decisions, and some guilds may limit leadership roles to residents for legitimacy.

### How do we handle a founder leaving?
If a founder leaves, remaining admins should:
- Ensure at least one admin remains.
- Update the guild's "founding members" list (for historical accuracy).
- If no admins remain, contact CivicNotes support for assistance.

### What's the best way to thank departing members?
A brief, kind message acknowledging their contributions (if they left positively) leaves the door open for return and maintains goodwill.
